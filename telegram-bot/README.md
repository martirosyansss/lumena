# MedBridge Tourism — Telegram-бот для привлечения клиентов

Лид-генерирующий бот для медицинского туризма **РФ → Армения**. Знакомит клиента
с направлениями, считает ориентировочную смету, отвечает на частые вопросы и
собирает заявки — каждая заявка уходит **в Telegram менеджерам** и **в Google-таблицу**,
плюс дублируется в локальный файл, чтобы ничего не терялось.

- Языки: **русский / English / Հայերեն**
- Стек: **Node.js + Telegraf**, запуск через **long polling** (без домена и SSL)
- Цифры цен и калькулятор **синхронизированы с сайтом** (`index.html`)

## Быстрый старт

### 1. Установка

```bash
cd telegram-bot
npm install
cp .env.example .env   # на Windows: copy .env.example .env
```

Нужен **Node.js 18.17+** (используется встроенный `fetch`).

### 2. Заполнить `.env`

| Переменная | Обязательна | Что это |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен от [@BotFather](https://t.me/BotFather). |
| `MANAGER_CHAT_ID` | ★ | Чат, куда падают заявки (группа менеджеров или ваш личный chat id). |
| `SHEETS_WEBAPP_URL` | ★ | URL Web-app из Google Apps Script (см. ниже). |
| `SHEETS_SECRET` | ★ | Секрет, совпадающий с `SECRET` в `Code.gs`. |
| `COORDINATOR_USERNAME` | — | @username координатора (без `@`). |
| `COORDINATOR_WHATSAPP` | — | Номер WhatsApp, только цифры (`79000000000`). |
| `COORDINATOR_PHONE` | — | Телефон для отображения. |
| `GUIDE_URL` | — | Ссылка на PDF-гайд. |
| `SITE_URL` | — | Адрес сайта. |
| `DEFAULT_LANG` | — | `ru` / `en` / `hy` (по умолчанию `ru`). |

★ — не обязательны для запуска, но без них заявки не доставляются (останется
только локальная резервная копия `data/leads.jsonl`).

### 3. Узнать `MANAGER_CHAT_ID`

- **Группа менеджеров:** создайте группу, добавьте туда бота, напишите любое
  сообщение, затем откройте в браузере
  `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates` и скопируйте
  `"chat":{"id":-1001234567890}` (у групп id отрицательный).
- **Личка:** напишите боту `/start`, откройте тот же URL и скопируйте свой `id`.

### 4. Подключить Google-таблицу

1. Создайте Google-таблицу.
2. **Расширения → Apps Script**, вставьте код из
   [`google-apps-script/Code.gs`](google-apps-script/Code.gs).
3. В `Code.gs` задайте `SECRET` и впишите то же значение в `.env` как `SHEETS_SECRET`.
4. **Развернуть → Новое развёртывание → Веб-приложение**: «Запуск от имени: Я»,
   «Доступ: Все». Скопируйте URL в `.env` как `SHEETS_WEBAPP_URL`.
5. После любой правки скрипта — переразверните развёртывание.

> Без этого шага бот работает, но в таблицу не пишет (оставьте `SHEETS_WEBAPP_URL` пустым).

### 5. Запуск

```bash
npm start      # боевой запуск
npm run dev    # с авто-перезапуском при правках (node --watch)
```

В консоли появится `[bot] @<имя_бота> is up (long polling)`.

## Деплой

Бот работает на long polling — домен и SSL не нужны.

**Railway / Render:**
- Создайте сервис из репозитория, корень — папка `telegram-bot`.
- Build: `npm install`, Start: `npm start`.
- Переменные окружения — из таблицы выше (раздел Variables/Environment).

**VPS (через pm2):**
```bash
npm install -g pm2
pm2 start src/bot.js --name medbridge-bot
pm2 save && pm2 startup
```

**Docker:**
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --omit=dev
COPY . .
CMD ["node", "src/bot.js"]
```

> Запускайте только **один** экземпляр бота на один токен — иначе Telegram вернёт
> ошибку `409 Conflict` (две поллинг-сессии конфликтуют).

## Безопасность

- **`.env` и `data/` в `.gitignore`** — токен и заявки не попадают в git.
- Если токен бота где-то «засветился», отзовите и пересоздайте его в @BotFather
  командой `/revoke`, затем обновите `BOT_TOKEN`.
- `SHEETS_SECRET` защищает Web-app от посторонних записей — держите его непустым.

## Медицинский дисклеймер

Бот не оказывает медицинских услуг и не гарантирует результат лечения. Во всех
ценовых блоках выводится: «Имеются противопоказания, необходима консультация
специалиста». Цены — рыночные ориентиры из открытых источников, не оферта.
